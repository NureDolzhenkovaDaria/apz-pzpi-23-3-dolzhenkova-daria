3 ОПИС ВИКОНАНОЇ РОБОТИ

Для реалізації мобільного клієнта системи «EmoAd» було використано мову розробки Kotlin та фреймворк інтерфейсу Jetpack Compose. Застосунок має головне вікно користувача та панель адміністратора.
3.1 Стек технологій та інструменти розробки:
-	Платформа: Android API 34 (Android 14).  
-	Мова програмування: Kotlin (версія 1.9).  
-	Інструмент збірки: Gradle.  
-	Мережева взаємодія: бібліотека Retrofit 2 для побудови REST-клієнта та асинхронного виконання HTTP-запитів.
-	Формат обміну даними: JSON (конвертер Gson).  
-	Генерація контенту: бібліотека ZXing (BarcodeEncoder) для динамічного перетворення промокодів у графічні QR-коди.
3.2 Структура та логіка роботи клієнтської частини:
Користувацький інтерфейс складається з двох основних екранів:
Головний екран: містить матрицю кнопок-емоджі для вибору настрою, відображення слогана та QR-код. 
Панель адміністратора: інтерфейс із авторизацією за ПІН-кодом, який реалізує створення, перегляд таблиці, редагування та видалення записів реклами в базі даних SQLite бекенду.
При натисканні користувачем на кнопку емоджі застосунок формує асинхронний POST-запит із JSON-тілом до центрального Flask-сервера за допомогою Retrofit. Отриманий у відповідь JSON-пакет розбирається конвертером Gson, після чого застосунок миттєво оновлює текстовий слоган на екрані та за допомогою модуля ZXing кодує текстовий промокод у QR-код. При додаванні, зміні чи видаленні реклами в адмін-панелі, застосунок надсилає відповідні HTTP-запити (POST, PUT, DELETE), після чого автоматично викликає функцію refreshAds() для миттєвого перемальовування таблиці актуальними даними з сервера.

5 ВИКОРИСТАННЯ ШІ ПІД ЧАС ВИКОНАННАЯ РОБОТИ

1.	Проєктування архітектури та логіки станів 
Промпт: «Я хочу зробити мобільний застосунок на Jetpack Compose та Flask. Застосунок має два меню: користувацький (вибір емоції, асинхронний запит через Retrofit, генерація QR-коду за допомогою ZXing, обробка помилок мережі) та адміністративне (вхід за ПІН-кодом, CRUD-операції в базі даних SQLite).
Результат генерації ШІ: допоміг чітко розділити логіку застосунку на три стани для діаграми: початкове очікування в меню, клієнтську гілку (надсилання запиту через Retrofit - обробка відповіді та створення QR-коду через ZXing - показ акції) та адміністративну гілку (перевірка ПІН-коду - перехід до керування базою даних SQLite в реальному часі). 
2.	Генерація UI-шаблону на Jetpack Compose (Single Activity)
Промпт: «Згенеруй структуру головного екрана мобільного клієнта на Jetpack Compose мовою Kotlin. Необхідно реалізувати пастельний фон (колір #F1EAFF), адаптивну матрицю з 4 кнопок-емоджі (Веселий, Сумний, Втомлений, Злий) з однаковою вагою в рядах, текстове поле для виведення динамічного слогана та контейнер для відображення згенерованого Bitmap QR-коду. Організуй код у межах одного файлу MainActivity».
Результат генерації ШІ: Базовий каркас функції @Composable fun MainScreen(), який використовує елементи Box, Column, Row та інструмент Modifier.weight(1f) для створення адаптивної сітки кнопок. Також згенеровано логіку умовного рендерингу екранів через змінну стану currentScreen ("main" / "admin").
3.	Налаштування клієнт-серверної взаємодії через Retrofit 2
Промпт: «Створи інтерфейси Retrofit 2 для зв'язку Android-клієнта з Flask-сервером. Потрібно реалізувати два сервіси: EmoAdApiService (для надсилання POST-запиту з емоцією користувача та отримання слогана й промокоду) та AdminApiService (для виконання повного циклу CRUD: GET для списку, POST для додавання, PUT для оновлення за ID та DELETE для видалення). Додай відповідні Data-класи для парсингу JSON через Gson».
Результат генерації ШІ: Готові інтерфейси з HTTP-анотаціями та супутні моделі даних (AdResponse, AdItem, SimpleResponse) з підтримкою анотацій @SerializedName.
4.	Оптимізація обробки даних на Flask-сервері (Синхронізація JSON)
Промпт: «Мій Flask-сервер повертає помилку 404 під час обробки POST-запиту від мобільного додатка. Сервер очікує параметри з URL (request.args), а Retrofit надсилає дані в тілі запиту як JSON. Перепиши ендпоінт /api/get_ad на Python так, щоб він коректно приймав JSON-боді, синхронізуй назви ключів (slogan та promo_code) з Kotlin-моделями та забезпеч збереження змін у базі SQLite через conn.commit()».
Результат генерації ШІ: Модифікований Python-код для Flask, який використовує метод request.get_json() замість request.args.get(). Впроваджено сортування ORDER BY RANDOM() LIMIT 1 для забезпечення видачі випадкової реклами, якщо в базі є кілька варіантів для однієї емоції, та додано обов'язкові виклики conn.commit() для синхронного збереження CRUD-операцій на диску.

354 private fun sendEmotion(emotion: String, apiService: EmoAdApiService, context: android.content.Context, onResult: (String, Bitmap?) -> Unit) {
355     apiService.getAdByEmotion(EmotionRequest(emotion)).enqueue(object : Callback<AdResponse> {
356         override fun onResponse(call: Call<AdResponse>, response: Response<AdResponse>) {
357             if (response.isSuccessful && response.body() != null) {
358                 val ad = response.body()!!
359                 try {
360                     val encoder = BarcodeEncoder()
361                     val bitmap = encoder.encodeBitmap(ad.promo_code, BarcodeFormat.QR_CODE, 400, 400)
362                     onResult(ad.slogan, bitmap)
363                 } catch (e: Exception) { e.printStackTrace() }
364             } else {
365                 Toast.makeText(context, "Рекламу не знайдено", Toast.LENGTH_SHORT).show()
366             }
367         }
368         override fun onFailure(call: Call<AdResponse>, t: Throwable) {
369             Toast.makeText(context, "Помилка зв'язку з сервером", Toast.LENGTH_SHORT).show()
370         }
371     })
